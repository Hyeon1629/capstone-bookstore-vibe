## 슬라이드 1 · 프로젝트 한 줄 소개

### **숨은 책방** — *동네 책방을 위한 포켓몬 고*

> 지도에서 동네 책방·도서관·북카페를 **발견**하고, GPS로 **방문 인증**하고, 내 동네 책방 지도를 **수집**하는 모바일 앱.

**가치 축 2개**
1. **발견 (Discovery)** — "동네에 책방이 이렇게 많았다" 의 와우 모먼트
2. **수집 (Collection)** — 방문 지도가 한 점씩 채워지는 시각적 만족감

## 슬라이드 15 · 트러블슈팅 (1) — 환경 이슈

### 카카오 SDK `kakao=false maps=false load=false`
- **원인 1**: Vite dev 서버 포트가 5173 이 아닌 5180 으로 자동 이동 → 카카오 콘솔에 미등록 도메인 → 401
- **해결**: `vite.config.ts` 에 **`server.strictPort: true`** 추가 → 포트 고정
- **원인 2**: Edge/Brave 의 **Tracking Prevention** 이 `dapi.kakao.com` 차단
- **해결**: 추적 방지 예외 추가 또는 Chrome 권장

### React Router 경고 + Geolocation Violation
- `BrowserRouter future={{ v7_startTransition, v7_relativeSplatPath }}` 로 silence
- `useGeolocation` 에 **Permissions API** 추가 → 권한 `granted` 일 때만 `watchPosition` 호출 (Chrome 정책 준수)

## 슬라이드 16 · 트러블슈팅 (2) — 백엔드 이슈

### Firestore 복합 인덱스 요구
- `fetchUserVisits` 의 `where userId == ... + orderBy visitedAt desc` 조합이 복합 인덱스 필요
- **해결**: `orderBy` 제거하고 **클라이언트 측 정렬** — 인덱스 없이도 작동
- `hasRecentVisit`, `fetchMoodTags`, `fetchAllUserMoodTags` 도 동일 패턴 적용

### 카카오 영업시간 가져오기 (시도)
- HTML 스크래핑 → 카카오 페이지가 SPA 라 HTML 에 영업시간 데이터 없음
- 카카오 내부 API `places/panel3/{id}` 발견 → JSON 으로 영업 상태 + 오늘 시간 + 휴무일 풍부하게 제공
- **현재 제약**: 공개 CORS proxy 의존이라 시연 환경에서 안정성 100% 보장 어려움
- **시연 시 대응**: 영업 정보 없는 경우 `영업 정보 없음` 으로 깔끔하게 fallback

## 슬라이드 17 · 시연 동선 (10단계)

```
1. 온보딩 3슬라이드 (Framer staggered fadeUp)
2. 회원가입 (실시간 검증 + 닉네임 중복 → 대안 칩)
3. 지도 홈 → 시드 5곳 + 카카오 실시간 +N 곳
4. 카테고리 토글 (주황 ↔ 초록)
5. "서강" 검색 → 두 도서관 핀 강조 ⭐
6. 핀 탭 → 미리보기 시트 → [상세 →]
7. 책방 상세 (카카오 실제 사진 + 분위기 태그 9+3 + [길찾기] 카카오맵 새 탭)
8. 마이페이지 → 시연 모드 ON + [더미 방문 채우기] 1회
9. 지도로 돌아가 핀 long-press → radar → paper → 황금 스탬프 모달 → 이모지 선택 ⭐
10. /bookshelf → 통계 카운트업 + NO.07 책 등 spine 리스트 + 방문 지도 골드 핀
```
