# README_TEST.md — 수동 테스트 시나리오

본 문서는 각 STEP 종료 시 사용자가 직접 클릭해서 확인할 수동 테스트 시나리오를 누적한다.

## 사전 준비

### Firebase 콘솔 (이미 완료된 항목 외)

1. **Authentication → Sign-in method → Email/Password 활성화**
2. **Firestore Database 생성**
   - 위치: `asia-northeast3` (서울)
   - 시작은 **테스트 모드** (30일간 모든 read/write 허용)
3. 테스트 모드 종료 후엔 보안 규칙 적용. 과제 진행 동안은 테스트 모드 유지 가정.

### 개발 서버

```bash
npm run dev
# → http://localhost:5173
```

## STEP 2 — 인증 + 온보딩

### 시나리오 A. 신규 회원가입

1. 시크릿 창에서 `http://localhost:5173` 진입
2. 자동으로 `/onboarding` 으로 이동 (3개 슬라이드)
3. 인디케이터 도트 클릭으로 슬라이드 이동, [다음] 버튼으로 진행
4. 마지막 슬라이드 [시작하기] → `/signup` 이동
5. 이메일 입력 → 실시간 형식 검증 (OK / Error)
6. 비밀번호 입력 → 6자 미만 시 카운터 (`3/6`), 6자 이상 시 OK
7. 닉네임 입력:
   - 1자: 형식 오류
   - `서연` 입력 → "확인 중..." → "사용 가능한 닉네임이에요" (300ms 후)
8. [가입하고 시작하기] 버튼 활성화 → 클릭
9. OS 위치 권한 모달 표시 (허용/거부 선택)
10. `/map` placeholder 진입 → UID / email / 닉네임 / fallback area 표시 확인

### 시나리오 B. 닉네임 중복 검증

1. 시나리오 A 로 `서연` 가입 후 로그아웃 (`/map` 의 로그아웃 버튼)
2. 시크릿 창 새로 띄워 다시 `/signup` 진입
3. 동일 닉네임 `서연` 입력 → 300ms 후 "이미 사용 중이에요" + 대안 칩 (`서연_2`, `서연1`, `서연a`)
4. 대안 칩 클릭 시 입력값 자동 교체

### 시나리오 C. 로그인 + 자동 로그인

1. 가입한 계정으로 로그아웃 후 `/login` 진입
2. 잘못된 비밀번호로 시도 → "이메일 또는 비밀번호가 일치하지 않아요"
3. 올바른 비밀번호로 로그인 성공 → `/map`
4. 브라우저 닫고 다시 `http://localhost:5173` 진입
5. **자동 로그인되어 `/map` 으로 바로 이동 확인** (`onAuthStateChanged` + browserLocalPersistence)

### 시나리오 D. 온보딩 1회 노출

1. 시나리오 A·C 마치면 `localStorage.onboardingSeen = 'true'` 설정됨
2. 로그아웃 후 `http://localhost:5173` 진입
3. `/onboarding` 이 아닌 **`/login`** 으로 바로 이동 확인
4. (재현하려면 DevTools → Application → Local Storage → 해당 키 삭제)

### 시나리오 E. 약관·비밀번호 찾기 링크

1. 회원가입 화면의 [이용약관] / [개인정보처리방침] 클릭
2. "v1에서 제공 예정" alert 표시
3. 로그인 화면의 [비밀번호 찾기] 클릭 → 동일 alert

### 시나리오 F. 라우트 가드

1. 로그아웃 상태에서 `http://localhost:5173/map` 직접 진입
2. 자동으로 `/login` 으로 redirect
3. 로그인 상태에서 `http://localhost:5173/login` 직접 진입
4. (현재는 `/login` 페이지 그대로 표시 — STEP 3 에서 가드 강화 검토)

## STEP 3 — 지도 홈

2. 현재 위치 주변 책방·도서관·북카페 핀 표시 (카카오 Local 검색) — 핀은 SVG drop shape + 카테고리 색 + 글리프(book/cup)
3. 헤더에 "반경 안에 책방 N곳" (lamp 색 강조)
4. 카테고리 칩 [서점·헌책방] 탭 → 주황 핀만 남고 초록 핀은 숨김
5. [도서관·북카페] 탭 → 초록 핀만
6. [전체] 탭 → 전체 복원

### 시나리오 H. 책방 이름 검색 강조

1. 헤더 검색바에 표시된 책방 이름의 일부를 입력
2. 매칭 외 핀은 opacity 0.3 으로 dim
3. 이름이 매칭되는 책방만 선명하게 남음
4. 검색 첫 결과의 좌표로 자동 setCenter
5. 검색바 우측 ✕ 버튼으로 초기화

### 시나리오 I. 핀 탭 → 미리보기 시트 → 상세

1. 임의의 책방 핀 탭
2. 핀 focused 상태 (글로우 + scale 1.08 + translateY -4px)
3. 하단 미리보기 시트가 slide-up (250ms, translateY 20→0)
4. 시트에 책방명 / 카테고리 / 거리 표시
5. 시트 [상세 →] 클릭 → `/bookstore/{id}` placeholder 페이지로 이동 ("STEP 4 예고")
6. 뒤로가기 → 지도 홈 유지

### 시나리오 J. 검색 0건

1. 검색바에 "xyz_없음" 입력
2. 모든 핀이 dim 처리
3. 지도 중앙에 "근처에 일치하는 책방이 없어요" floating 메시지

### 시나리오 K. 현재 위치 FAB

1. 사용자 위치 권한 허용 후 진입 시 `/map` 의 setUserLocation 자동 호출
2. 핀 시트 미오픈 상태에서 FAB 위치 bottom 24px
3. 시트 열린 상태에서 FAB 위치 bottom 168px (250ms 트랜지션)
4. FAB 클릭 시 사용자 좌표로 setCenter

### 시나리오 L. BottomNav 탭 전환

1. 하단 [북쉘프] 탭 → `/bookshelf` placeholder (STEP 6 예고)
2. 하단 [마이] 탭 → `/mypage` placeholder + 로그아웃 버튼
3. [지도] 탭 → `/map` 복귀
4. 활성 탭의 글리프는 filled, 비활성은 outline

## STEP 5 — GPS 인증 + 분위기 입력

### 사전 조건

**Firestore 인덱스 생성** (첫 인증 시도 시 콘솔이 안내):

1. 첫 인증 시도 시 콘솔(Devtools) 에 `The query requires an index` 에러 + 자동 생성 링크 출력
2. 링크 클릭 → Firebase 콘솔에서 [생성] 클릭 → 1~2분 대기
3. 인덱스 활성화 후 다시 시도

(또는 미리 Firebase Console → Firestore → 색인 → `visits` 컬렉션 + `userId Asc, bookstoreId Asc, visitedAt Desc` 복합 색인 수동 추가)

### 시나리오 R. GPS 자동 인증 ⭐

1. 책방 50m 이내 + 5초 체류 시 자동 인증 트리거 (실 모바일 테스트 환경)
2. 화면에 **radar pulse** 풀스크린 2초간 표시 (info 컬러 3개 ring 확장)
3. **paper fadein** 250ms (방문 확인 신호)
4. **MoodInputModal** 진입: 황금 스탬프(점선 회전 ring 12s) + 7개 스파클 + `STAMP COLLECTED · #001`
5. 5개 이모지 중 `🤫 한적한` 선택 → 카드 lamp 보더 + `translateY -2px`
6. [북쉘프에 기록하기] 클릭 → Firestore visits + moodTags 추가 → `/bookshelf` 이동

### 시나리오 S. 같은 날 중복 방지

1. 시나리오 R 직후 같은 날 같은 책방에 재진입
2. radar pulse 없이 상단 토스트 "이미 방문 인증한 책방이에요" 2.5초 표시
3. 모달 표시 안 됨

### 시나리오 T. 분위기 모달 닫기

1. 시나리오 R 의 모달 표시 상태에서 키보드 `ESC` 눌러 닫기
2. visits/moodTags 모두 저장 안 됨 (트리거 취소 효과)
3. 배경 클릭은 무시 (5 이모지 선택 유도)
4. [건너뛰기] 클릭 시 — visits 만 저장되고 moodTag 는 저장 안 됨 → `/bookshelf` 이동

### 시나리오 X. 누적 통계 + 카운트업 ⭐

1. GPS 인증으로 책방 2~3곳 방문 기록
2. `/bookshelf` 진입 → 누적 통계 카드의 숫자가 0 → 실제값까지 약 0.8초 카운트업
3. 좌측 누적 방문 (lamp 색 강조), 우측 이번 달 (paper 색)
4. 카드 상단에 누런 띠 1px 보일 듯 말 듯 (책 등받이 메타포)

### 시나리오 Y. 방문 지도 미니뷰 + 책 등 리스트

1. `/bookshelf` 의 방문 지도 SVG: 방문 책방은 컬러 핀(드롭쉐도우 글로우), 미방문은 회색 점
2. 좌하단 legend: `● 방문 N` + `● 미방문 N`
3. 아래 방문 기록: 책 등 spine 모티프 (36×44, NO.01, NO.02 ... Gowun Batang)
4. 분위기 이모지가 있으면 책방명 옆에 표시 (방문 인증 시 선택한 이모지)
5. 리스트 항목 staggered fade-in (40ms 간격)
6. 항목 클릭 → `/bookstore/:id` 상세

### 시나리오 Z. 마이페이지 프로필 카드

1. `/mypage` 진입
2. 프로필 카드: 아바타(닉네임 첫 글자 + lamp 그라데이션 + glow) + 닉네임 + 이메일 + `★ EXPLORER · LV.N`
3. 누적 방문 3개 미만 → `START`, 1~6개 → `LV.1`, 7~12개 → `LV.2`, 13개 이상 → `LV.3`
4. 하단 MiniStat 2열: 누적 방문 / 이번 달 (카운트업)
5. 메뉴 그룹: [설정] (알림·앱 정보 - 클릭 시 토스트) / [계정] (로그아웃)
6. 푸터: `숨은 책방 · v1.0.0`

### 시나리오 AA. 로그아웃 다이얼로그

1. `/mypage` [로그아웃] 메뉴 (error 색) 클릭
2. 풀스크린 scrim + 중앙 카드 모달 (`로그아웃 하시겠어요?`)
3. ESC / 배경 클릭 / [취소] 클릭 모두 모달 닫기
4. [로그아웃] 클릭 → Firebase signOut → `/login` 으로 redirect
